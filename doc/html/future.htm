Work in Progress & Future Plans

This page list the work in progress and future plans. The activities are described in priority orders. Work in progress are likely to be implemented and the ambition is to implement future plans. However since this is open source project currently without any intressents the tempo is unpredictable.

Work In Progress

  • 2022-01-29, starting the developing of a crypto gateway (bitcoin/ethereum) allowing participants to deposit and re-draw coins from the position controller deposit
  • 2022-01-29, pre-trading validation is by today added. A position controller component has been added. If pre-trading validation is configured, buy order are validated against the cash holdings and the exposure in the market. Sell orders are validated against the account inventory and the exposure inthe market. The position is updated in real-time as trading takes place.

Future Plans / Work

  • Realtime Pre-Trading Limit Handling, configureable real-time settlement / limit controll is to be implemented i.e DVP This will enable the system to verify in real-time as part of trade execution and order entry that client capacity if available. This also include implementation of crypto wallets.
  • Partitioning, even if the design allow function (on service) and data (market, symbol) partitioning the solution is currently packed up in as one service. Needs to be refactored configuration wise.
